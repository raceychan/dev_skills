# Spec vs Todo

## Use a tech spec for
- System design
- Change boundaries
- Architecture decisions
- Tradeoffs
- Risks and compatibility

## Use a todo list for
- Execution order
- Atomic tasks
- Acceptance criteria
- Delivery checkpoints
- Parallelization and blockers

## Rule of thumb
If the document answers "what changes in the system and why", it is a spec.
If it answers "what do we do next, in what order, and how do we know each step is done", it is a todo list.
